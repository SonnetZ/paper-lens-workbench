# Reading Cockpit Design

## Goal

Paper Lens should feel like a reading cockpit: the paper is the primary surface, and review tools behave like quiet instruments around it. Minimalism here means hierarchy, restraint, and contextual controls, not a stack of white cards and forms.

## Direction

Use an Apple-like, technical minimal language: soft neutral surfaces, thin rules, precise typography, a single red accent, and task modes instead of visible tool sprawl. The center reader stays dominant. The left queue is an index rail. The right workspace is a mode-based instrument panel.

## Layout

- Reading column owns the visual center and includes a compact cockpit bar.
- Cockpit bar shows current record, source availability, evidence count, model source, and selected knowledge base.
- Queue remains collapsible and reads as a document index rather than a card list.
- Workspace uses task modes: `Assist`, `Evidence`, and `Review`.
- Evidence tray becomes an evidence stack with a compact collapsed state and timeline-like saved items.

## Interaction

- `Assist` contains model source, ask, brief, and knowledge base tools.
- `Evidence` contains evidence status and knowledge base operations.
- `Review` contains screening, extraction, and review material export.
- The cockpit bar provides orientation without adding another form.
- Selection assistant remains anchored near selected text and keeps translate, save, and ask actions.

## Visual Rules

- Avoid cards inside cards.
- Prefer bands, rails, inline summaries, thin separators, and task modes.
- Keep rounded corners tight.
- Preserve accessibility: semantic regions, visible focus, keyboard-operable controls.
- Do not add decorative gradients, orbs, or marketing hero composition.
